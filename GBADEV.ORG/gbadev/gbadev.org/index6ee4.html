no$gba v1.9 freeware
By SimonB [Tuesday July 13th 21:53:02 UTC 2004]		 News : General
Permalink
Martin Korth writes:

Heya!

Big change to the unexpected nocash marketing concept:
no$gba is now freeware for gamers! http://work.typo3.work.de/nocash/gba.htm

The freeware doesn't include any debugging functions, but it's including the complete GBA emulation part, for free. I hope that will satisfy everybody, including piracy sceners, maybe it'll increase the popularity of no$gba, or maybe even convince some people that emulated hardware isn't necessarily incompatible to real hardware ;-) Or otherwise, maybe some users let me know if they find any unknown compatibility problems.

Anyways, as a mini debugger, the freeware version includes a silly window showing the error statistics & quality rating for the emulated game. It is counting obvious programming bugs (such like attempts write into ROM, to access mis-aligned memory addresses, etc.) It's also adding 100 penalty bugs if the program doesn't support low power battery saving, the final rating is then:
0 bugs at all - perfect
1-199 bugs per second - good
200-999 bugs per second - medium
1000 or more bugs per second - disqualified
That is, of course, not very objective since a single bug which is repeated thousands of time per second would result in a lower rating as for hundreds of bugs which are executed only once per second.

There'll be also a low-cost shareware version for hobby programmers with (almost) complete debugging functions (hoping that commercial users won't use it at work, except eventually for testing no$gba). And, I've made a new price list, which is now also including single user licences for commercial use, so that smaller companies are no longer restricted to use hardware debuggers.

The actual no$gba debug version webpage is now cleverly hidden between dozens of useless links on the freeware page - I hope programmers will still find it, whilst everybody else should be successfully convinced that there isn't anything of interest to him, which is true, aside from downloading the freeware version.

The new debugger version is having two bug-fixes, some new details, and I've been working on demangling c++ labels, eg. the label "Function__6Methodv" will be displayed as "Method::Function(void)". That's at the moment working in Profiler/tree and Alt+L windows only, and the latter one is still alphabetically sorted by the mangled names, hope to get that fully implemented in next update(s).

Ciao, Martin
